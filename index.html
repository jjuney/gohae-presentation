느낀점 12시간 후기 — 장점 & 단점

장점

  • 원하는 기능을 빠르게 구현 — 아이디어 → 동작하는 서비스까지 12시간
  • FE · 인프라 등 약한 영역도 Claude가 보조 → 혼자서 풀스택 가능
  • 기획 · 문서 · UI · 코드를 하나의 흐름으로 처리 — 컨텍스트 유지가 핵심
  • 코드 짜는 게 재밌어짐 — 개발에 대한 흥미 · 몰입도 상승

단점 / 주의점

  • 실제 프로덕션 배포엔 리스크 존재 — 보안 · 성능 · 엣지케이스 검토 필요
  • Claude가 틀릴 수 있음 — 정확한 도메인 지식이 있어야 시너지 극대화
  • 대충 알고 쓰면 대충 나옴 — 질문의 질이 결과물의 질을 결정
  • "빠르게 만들기"와 "잘 만들기"는 다름 — MVP 이후 리팩토링 필수
